Les directives AngJS :

* Les directives permettent d'étendre HTML avec de nouvelles attributs, appelé Directive.
* On peut définir nos propres directive, afin d'ajouter de nouvelles fonctionnalités à notre application.

Quelques directives :

* ng-app : une directive permetant de décalarer une app AngJS.
* ng-init : une directive permetant d'initialiser des variables, des objets ...
* ng-model : une directive permetant de lier des champs HTML. (voir cette directive comme une déclaration de variable)

Création de nouvelles directives :

On crée une nouvelle directive avec le code suivant (.directive)

var app = angular.module("myApp", []);
app.directive("w3TestDirective", function() { return { template : "Made by a directive!" }; });
- Lors de la définition d'une directive, elle doit être écrite en camelCaseForm.
- A son invocation, elle doit être écrite avec (-). exemple : w3-test-directive. Elle peut être utilisée en tant que :
     * Balise.
     * Attribut.
     * Class.
     * Commentaire.


Exempel :
- < w 3-test -directive> < /w 3 -test -directive> :

Les restrictions :

On peut réstreindre l'utilisation des directives à des balises, attributs, class ou commentaires, en rajoutant restrict : "A", comme ceci :
  • E : for Element name
  • A : for Attribute
  • C : for Class
  • M : for Comment
     * return { restrict : "A", template : "______" };
